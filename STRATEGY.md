# Курс — трекер приёма лекарств. Стратегия

*03.07.2026. Side-project в семействе Дельта/Выдох: local-first, без сервера, приватность по умолчанию.*

## 1. Конкурентный анализ (сжато)

| Игрок | Сильное | Слабое (наша щель) |
|---|---|---|
| **Medisafe** (~11M польз.) | База лекарств, взаимодействия, «Medfriend» (уведомить близкого) | Перегружен, англо-центричен, аккаунт обязателен, монетизация давит |
| **MyTherapy** | Схемы любой сложности + дневник здоровья (давление, вес) | Тот же комбайн: онбординг долгий, данные в облаке (Германия) |
| **«Мои таблетки»** (РФ) | Гибкие схемы, перенос приёма, история | Устаревший UI, Android-центричен, нет вменяемой аналитики |
| **Mr. Pillster, Pills.Kit и пр.** | Просто напоминалка | Будильник без трекинга: нет adherence, курсов, остатков |
| **Заметки/будильник** (главный конкурент) | Ноль трения | Нет истории, нет «принял или нет?», нет курса |

**Вывод.** Рынок делится на «комбайны с аккаунтом» и «тупые будильники». Пусто посередине: **приватный трекер курса** — поставил курс за 30 секунд, отмечаешь одним тапом, видишь дисциплину приёма и остаток таблеток. Ровно ниша Дельты, но в здоровье.

**Позиционирование:** «Курс — довести курс лекарств до конца. Без аккаунта, данные не покидают телефон».

## 2. Roadmap

- **P0 — MVP (готов, эта сессия):** PWA. Аптечка (лекарства, дозы, время, дни недели, длительность курса, остаток), экран «Сегодня» с отметкой одним тапом, adherence % и стрик, история 4 недели, локальные уведомления, офлайн, всё в localStorage.
- **P1 — Надёжные напоминания (1 сессия):** Telegram-бот как канал доставки: PWA генерит расписание → бот шлёт пуши и принимает «✓ Принял» прямо из чата. Хостинг: Cloudflare Workers (бесплатно). Это закрывает главную слабость PWA на iOS.
- **P2 — Коуч (1–2 сессии):** ИИ-разбор через Claude API (как в Дельте): «ты стабильно пропускаешь вечерний приём по пятницам», прогноз «таблеток хватит до 12 июля — купи заранее». Экспорт истории врачу (PDF/CSV).
- **P3 — Близкие:** режим «уведомить близкого при пропуске» (киллер-фича Medisafe, дёшево реализуется через того же TG-бота).

## 3. PWA vs Telegram-бот — ресурсы

| | **PWA** | **TG-бот** |
|---|---|---|
| Разработка | ~1 сессия, 1 файл, готово сейчас | ~1 сессия + BotFather + деплой |
| Инфраструктура | 0 ₽, статика (GitHub Pages) | Хостинг с cron: CF Workers 0 ₽ / VPS ~200 ₽/мес |
| Надёжность напоминаний | **Слабое место:** на iOS пуши только у установленной PWA и без сервера нестабильны | **100% доставка** — пуш Телеграма всегда доходит |
| UX трекинга | Полноценный: экраны, история, аналитика | Ограничен чатом: кнопки, текст |
| Приватность | Идеал: данные не покидают устройство | Данные проходят через сервер |
| Хранение состояния | localStorage, бесплатно | Нужна БД (KV/D1) |

**Итог:** по чистой надёжности напоминаний бот выигрывает, по продукту (трекинг, аналитика, приватность) — PWA. Правильный ответ — не «или», а последовательность: **PWA сейчас (P0) + бот как канал доставки (P1)**. Суммарно ~2 сессии и 0 ₽ инфраструктуры.

## 4. Метрики успеха (свои, не чужие)

- Активация: первый курс заведён ≤ 60 сек от открытия.
- Retention-прокси: adherence ≥ 80% на второй неделе.
- North star: % курсов, доведённых до конца.
